### Задание 4 - составление HTTP запросов

```http

GET / HTTP/1.1
Host: sandbox.usm.com
User-Agent: Кирилл Киров

```

User-Agent — это HTTP‑заголовок, который сообщает серверу кто именно отправляет запрос.

Он используется для: определения **браузера** (Chrome, Firefox, Safari), **устройства** (ПК, телефон, планшет), **ОС** (Windows, Linux, Android)

А также:
* аналитики и статистики
* адаптации сайта под устройство
* защиты от ботов и подозрительных запросов

Пример обычного User-Agent браузера:  
Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 Chrome/145.0.0.0


```http  
POST /cars HTTP/1.1
Host: sandbox.usm.com
Content-Type: application/x-www-form-urlencoded
Content-Length: 38

make=Toyota&model=Corolla&year=2020
```

Основные методы HTTP:

**GET** - получение данных  
запрашивает информацию с сервера (без изменения данных)

**POST** - отправка данных  
создание новых ресурсов (формы, регистрация, добавление данных)

**PUT** - полное обновление ресурса  
заменяет существующий объект целиком

**PATCH** - частичное обновление  
изменяет только указанные поля

**DELETE** - удаление ресурса  
удаляет данные на сервере

## Дополнительные методы:

**HEAD** - как GET, но без тела ответа
проверка существования ресурса или заголовков

OPTIONS - какие методы разрешены сервером
👉 используется для CORS и диагностики

**TRACE** - трассировка запроса  
отладка (почти не используется)

**CONNECT** - создание туннеля  
используется для HTTPS через прокси

```http
PUT /cars/1 HTTP/1.1
Host: sandbox.usm.com
User-Agent: Кирилл Киров
Content-Type: application/json
Content-Length: 50

{
  "make": "Toyota",
  "model": "Corolla",
  "year": 2021
}
```

**PUT**  - полностью заменяет ресурс целиком (нужно отправлять все поля),   
**PUT** полностью заменяет ресурс на сервере новым. Например, если на сервере был объект {a:1, b:2}, и выполнить PUT с данными {a:5}, то результатом будет {a:5} — поле b исчезнет.  

**PATCH** - частично изменяет ресурс (передаются только те поля, которые нужно обновить).  
**PATCH** частично обновляет ресурс. Например, если на сервере был объект {a:1, b:2}, и выполнить PATCH с данными {a:5}, то результатом будет {a:5, b:2} — изменится только указанное поле.
